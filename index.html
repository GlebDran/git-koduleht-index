issue1

MVP (Minimum Viable Product) on toote arendamise strateegia, kus luuakse toote minimaalne versioon, millel on ainult kõige olulisemad funktsioonid. Selle eesmärgiks on kiiresti turule jõuda ja koguda kasutajate tagasisidet, et edasist arendust suunata. MVP printsiipi saab IT projektides kasutada, et testida idee elluviimise potentsiaali ja vähendada arenduse riske. Alustamiseks tuleks keskenduda toote põhifunktsioonidele, mis pakuvad väärtust varajastele kasutajatele, ning iteratiivselt täiendada toodet tagasiside põhjal.

issue2

Lokaalse Git repo loomine:

  • git init

Lokaalse repo ühendamine kaugrepo'ga:

  • git remote add origin https://github.com/kasutajanimi/repo-nimi.git

Kaugrepo esmakordne üleslaadimine:

  • git add .
  • git commit -m "Esimene commit"
  • git push -u origin master
